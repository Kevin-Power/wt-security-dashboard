# WT 資安情資中心 (CISO Dashboard)

## 系統簡報 - 管理層報告

---

## 1. 系統定位

**一句話價值主張：**
> 將分散的資安數據整合為單一風險指標，讓管理層快速掌握企業整體資安態勢。

---

## 2. 解決的痛點

| 現況問題 | 本系統解決方案 |
|---------|---------------|
| 資安數據分散在多個系統 | 統一整合 4 大數據源 |
| 無法快速評估整體風險 | 0-100 單一風險指數 |
| 報表製作耗時費力 | 自動化同步 + 即時儀表板 |
| 難以追蹤趨勢變化 | 90 天歷史趨勢分析 |
| 缺乏優先處理依據 | 權重化風險排序 |

---

## 3. 核心功能亮點

### 3.1 整體風險指數 (Security Score Card)

```
整體風險 = KB4×20% + NCM×35% + EDR×30% + HIBP×15%
```

| 數據源 | 權重 | 評估內容 |
|--------|------|----------|
| **KB4** | 20% | 員工釣魚訓練表現、資安意識 |
| **NCM** | 35% | 網路設備漏洞、韌體更新狀態 |
| **EDR** | 30% | 端點威脅檢測、惡意程式警示 |
| **HIBP** | 15% | 企業帳號外洩監控 |

**風險等級分類：**
- 🔴 嚴重 (≥80)：需立即處理
- 🟠 高 (60-79)：優先關注
- 🟡 中 (40-59)：持續監控
- 🟢 低 (<40)：狀態良好

### 3.2 四大資安模組

#### KB4 釣魚風險監控
- 追蹤員工資安訓練成效
- 識別高風險用戶群組
- 部門/組織別分析

#### NCM 設備漏洞管理
- P0/P1/P3 優先級分類
- CVSS 風險評分
- 韌體版本追蹤

#### EDR 威脅檢測
- 即時警示追蹤
- 嚴重程度分類
- 處理狀態管理

#### HIBP 帳號外洩
- 自動監控企業郵箱
- 外洩事件通知
- 密碼重設追蹤

### 3.3 趨勢分析

- 14/30/90 天風險趨勢圖
- 週對週比較
- 各模組獨立趨勢

## 4. 技術優勢

### 效能優化
- ⚡ **API 響應 < 100ms**（緩存命中）
- ⚡ **原生 SQL 聚合查詢**（減少數據庫負載）
- ⚡ **批量同步處理**（每批 100 筆）

### 可擴展性
- 📊 可自訂風險權重
- 📊 模組化架構，易於新增數據源
- 📊 支援 Docker 部署

## 8. Demo 重點

向老闆展示時的建議順序：

1. **整體風險儀表板**
   - 展示單一風險指數
   - 說明各模組權重

2. **風險趨勢圖**
   - 展示過去 14 天變化
   - 說明異常點原因

3. **待處理項目**
   - P0 設備數量
   - 待處理 EDR 警示
   - 新發現帳號外洩

4. **實際操作**
   - 重新整理展示即時更新
   - 點擊各模組查看詳情

## 9. Q&A 準備

**Q: 數據多久更新一次？**
> A: 每 15 分鐘自動同步，也可手動觸發即時更新。

**Q: 風險權重可以調整嗎？**
> A: 可以，管理員可透過 API 動態調整各模組權重。

**Q: 系統安全性如何？**
> A: JWT 認證、API 速率限制、角色權限控制、安全 HTTP 標頭。

**Q: 未來可以加入其他數據源嗎？**
> A: 可以，系統採用模組化設計，易於擴展新數據源。

**Q: 如果系統故障，影響範圍？**
> A: 本系統為唯讀儀表板，不影響原有資安系統運作。
