## Slide 5: 風險評分模型 (SSC)

**標題:** Security Score Card - 統一風險指標

**圖表:** 圓餅圖或橫條圖顯示權重

| 模組 | 權重 | 評估面向 |
|-----|------|---------|
| KB4 | 20% | 員工資安意識 |
| NCM | 35% | 設備漏洞風險 |
| EDR | 30% | 威脅檢測狀態 |
| HIBP | 15% | 帳號外洩風險 |

**風險等級:**
- 🔴 80+ 嚴重
- 🟠 60-79 高
- 🟡 40-59 中
- 🟢 <40 低

**講稿重點:**
> 「我們設計了一個 0-100 的風險評分模型，
> 根據各模組的重要性分配權重，NCM 設備漏洞佔最高的 35%，
> 因為基礎設施的安全是最關鍵的。」

## 📝 簡報製作建議

### 視覺設計
- 使用公司品牌色系
- 每頁不超過 6 行文字
- 多用圖表、截圖
- 數字要大且醒目

### 時間分配 (15 分鐘版)
| 段落 | 時間 |
|-----|------|
| 開場 + 問題背景 | 2 分鐘 |
| 解決方案介紹 | 2 分鐘 |
| 功能展示 | 5 分鐘 |
| 效益分析 | 2 分鐘 |
| 未來規劃 | 1 分鐘 |
| Demo | 2 分鐘 |
| Q&A | 1 分鐘 |

### 簡報技巧
1. 開場先講痛點，引起共鳴
2. 數字要對比（之前 vs 現在）
3. Demo 要流暢，先演練 2-3 次
4. 準備離線截圖以防萬一
5. 結尾強調效益，留下印象
